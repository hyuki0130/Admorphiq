# AR25 — Generic State-Space BFS

> Movement game solved by generic BFS over frame-hashed states; same strategy passes both v1 and v2 unchanged.

## Observations

### Frame-level
- Player-like entity (distinct color) that moves in response to ACTION1–ACTION4
- Obstacles / walls as persistent pixels
- Goal state reached when level_up detected

### Action effects
- `ACTION1..4` map to cardinal movements
- `ACTION6` not needed
- State changes are deterministic for pressed direction

## Mechanics Hypothesis

- Movement puzzle: reach goal cell by navigating around walls
- State fully captured by frame hash (downsampled 64×64 → 16×16)

## Solution Pattern (frame-only, generalizes)

`strat_bfs_state_space` in [`src/admorphiq/planner/bfs_solver.py`](../../../src/admorphiq/planner/bfs_solver.py):
1. Hash current frame into a compact state key
2. BFS from current state, expanding by trying each available action
3. Deduplicate visited states
4. On goal signal (`level_up` in response), commit the path
5. Budget-capped (default 500K state expansions)

**Why it generalizes**: zero dependence on game internals. Works on any game whose state is mostly captured by the visible frame and whose actions are deterministic transitions. Same solver cleared v1 AR25 2/8 and v2 AR25 2/8 (identical result).

## Shape of Wins
