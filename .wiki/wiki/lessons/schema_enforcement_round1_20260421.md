# Round 1 schema enforcement arc

> Measurement log for the R7 round-1 bench. The operational rules it
> produced are hosted in
> [[../architecture#LLM-Output-Shape-Enforcement]] and
> [[../../schema#Frontmatter-Policy]]. This page preserves the four-
> version bench arc so future rounds know what each layer fixed and
> what it cost.

## The four-version arc (2026-04-21)

| Version | Change added | Envs cleared | Total levels | Hallucinations |
|---------|--------------|--------------|--------------|----------------|
| v1 | R7a/b/c/e baseline | 0/40 | **0** | 40/40 empty primary |
| v2 | + strip_frontmatter | ? | **13** | creative keys |
| v3 | + Ollama JSON schema `format` | 16/40 | **20** | 26/40 invalid names |
| v4 | + strategy-name enum + uniqueItems | 15/40 | **19** | **0/40** |

Baseline (pre-R7): 15/40 raw envs cleared, 36 levels, 12 hallucinations.

## What each version actually fixed

- **v1 collapse root cause.** R7b graph retrieval started pulling
  `games/<TITLE>.md` pages. Their YAML frontmatter (generated by
  `scripts/generate_wiki_game_pages.py` for tooling) ended up in the
  prompt. Qwen 8B, recency-biased, imitated the frontmatter key/value
  shape as its own output. 40/40 envs returned invalid primary.
- **v2 (frontmatter strip).** `wiki_retrieval.strip_frontmatter`
  removes the leading `---\n...\n---\n` from any retrieved page.
  Qwen now emits free-form JSON but picks creative key names
  (`"strategy"` instead of `"primary_strategy"`).
- **v3 (Ollama JSON schema).** `OllamaBackend.generate` gained a
  `json_schema` kwarg passed as the `format` body parameter (Ollama
  0.20.3 supports this). Decoder constrains output to match. Qwen now
  emits the right keys but emits strategy names not in the 67-strategy
  whitelist for 26/40 envs.
- **v4 (enum + uniqueItems).** `primary_strategy` and
  `fallback_stack.items` bound to the live whitelist as JSON Schema
  `enum`. Decoder physically cannot emit a non-whitelist name.
  `uniqueItems: true` prevents Qwen from padding `fallback_stack` with
  duplicates (seen in v3 on CD82: `click_color_order` four times).

## Remaining gap — baseline 29 vs round-1 19 levels

4 envs account for −22 of the net −17 level loss: FT09 −6, CD82 −6,
SB26 −8, AR25 −2. On each, baseline 8B (pre-R7) picked the
game-specific strategy as part of its fallback stack via lucky
unconstrained pattern-match; v4 Qwen with enum-constrained decoding
picks a generic click strategy and leaves the game-specific one out
entirely. The prompt contains a title-match rule (`prefer
<title>_frame_only in primary`) but Qwen does not reliably follow it
under 16KB wiki context — same mode as
[[selector_is_advisory_not_enforced_20260421]].

Round 2's candidate fix: post-process the Hypothesis to inject the
title-matching whitelist strategy into `fallback_stack` when one
exists. This is an **augmentation** of Qwen's pick, not an override.

## Follow-ups already landed

- `architecture.md` now hosts the four-rule output shape enforcement
  contract and the routing-rules-require-python-reinforcement principle.
- `schema.md` hosts the frontmatter-is-tooling-not-content rule.
- `selector.md` carries an "Enforcement note" warning that rules are
  advisory.

## Falsification

This lesson becomes obsolete if a future round on a stronger model runs
without any of the four layers and produces ≥29 levels (baseline parity)
on the 40-env suite with 0 hallucinations. That has not been measured.
